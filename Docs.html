Await

The 'await' function waits so many miliseconds.

await 1000

The code above will wait 1 second, or 1000 miliseconds.

This function can also read a variable's value:

var a 500
await a

Button

The 'button' keyword is used to create a Button.

button name text

The code above defines a button named 'name' and sets its text to the text value.

This creates a button on the screen.

When a button is created, it's default position is (0,0) and it's text is the text value. It's default color is black and default font size and weight is 9 and Regular

Button Click

The 'buttonClick' function plays a file whenever a button is clicked

buttonClick buttonName PathToFile

Here is an example of it being used:

button btn Click!
buttonClick btn ~/ButtonClickScript.ezcode

In the example above, a button is created and named btn. Its text is set to 'Clcik!' and, on the next line, it's set to play a script whenever it is pressed. This will work exactly like the 'playFile await file' function.

Color

The 'color' function is used to update the color of an object, textbox, label, or button.

color controlName r g b

The code above updates the color the specified control to the specified values (r, g, b).

The 'r', 'g', and 'b' values represent the red, green, and blue values of the color. Each number ranges from 0-255 and they can be variables.

If the control is an object, this will change the color of the object.

If the control is a label, this will change the font-color of the label.

If the control is a textbox, this will change the font-color of the textbox.

If the control is a button, this will change the back-color of the button

If Statements

The 'if' statement is used to inact a function aslong as two values agree.

if val1 mid val2 : code...

Here is an example of it being used:

var a 5
if a > 2 : printRaw 5 is greater than 2!

In the example above, a variable is created and given the value of 5. The if statement then decides if 5 is greater than 2. Since it is, it prints to the console. If it wasn't, it wouldn't do anything after the colon.

Here are the mids that can be used:

'>' - greater than '<' - less than '=' - equal to '!' - not equal to

This statement can be stacke on top of itself like so:

if 1 ! 2 : if 3 > 1 : print It\_Works!

If statements can also be used with text.

'>' - text length greater than '<' - text length is less than '=' - text is equal to '!' - text is not equal to

Here is an example of that:

var a hello
if a > 4 : if a ! help : printRaw 'A' is not equal to help and it's length is greater that 4 ('hello' length is 5).

List

The 'list' keywrod is used to declare a list of variables.

list new name : v1,v2,v3
list add name val
list equals name index val
list clear name

Here are the four modifiers:

new add equals clear

New

list new even_numbers : 2,4,6,8,10

The 'new' modifier declares a new list of variables. They can be numbers or text, but it all has to be the same type (all numbers or all text).

To call a variable from the list above, this is what to do:

even_numbers:2

This calls the 2 index from this list. Index's start with zero, so the second index is the third value. so 'even__numbers:2' calls '6'

When stating the values in a list, make sure to NOT put spaces between the commas. Everthing after the colon has to not have any spaces. Same with calling a single variable from a list. You have to put the list name, colon, and then number, not variable. If a variable needs to be the index, use the 'varEquals var_name FromList list_name index'

Add

list add even__numbers 12

This adds a value to the end of a list. There has to be a list already named 'even__numbers' and it has to be an all numbers list.

Equals

list equals even__numbers 3 9

This makes an index in a list equal to the specified value. In the code above, 3 is the index and 9 is the new value.

Clear

list clear even__numbers

This clears all of the values in the list.

List Raw

The 'listRaw' keywrod is used to declare a list of variables. It is neerly identical to 'list' with a few key differences

listRaw new name : v1,v2,v3
listRaw add name val
listRaw equals name index val
listRaw clear name

The main differences is that when declaring a list, special characters (\_,\!, and \n) will not work, calling variables with their values won't work, and the values can be mixed between numbers and text.

Loop

The 'loop' statement is used to loop through code so many times.

loop times
end

Loops need an end to finish on. They can also only be used one time per script.

Here is an example of it being used:

var a 0
loop 10
a + 1
print a
end

In the example above, a variable is created and given the value of 0. A loop statement then loops through adding one to it and printing the current value of it.

Once it is declared how many times it can be used, it will loop through it that many times.

This statement can be stacke on top of itself like so:

var a 100
loop a
a - 10
end

Even though the variable for the loop times is decreaseing by ten every loop, it will still loop through 100 times. the 'a' variable will be equal to -900 at the end of the loop.

Remember when using it, it can only be used once per script. If a loop is needed twice, play another script that contains that loop using the 'playFile now file' function.

Multi-Line

The 'multiLine' function sets a textbox to have multi-line capibility or not.

multiLine textboxName value

Here is an example of it being used:

textbox txt \!
multiLine txt true
write txt Hello\nWorld

In the example above, a textbox is created and named txt. It is than set to be multi-line capable and is given text that is multiple lines long.

Heres are some of the values it can be given.

TRUE: true True 1 Yes yes
FALSE: false False 0 No no

Object

The 'object' keyword is used to create an object.

object objectName x

The code above defines an object named 'objectName' and gives it the amount of sides determined by the value of 'x'.

Object creates a polygon on the screen.

object obj 5

This code declares a pentagon named 'obj'
The sides value has to be greater than 2. When an object is created, it's default position is (0,0) and has a size of (50,50). It's default color is black

Play File

The 'playFile' function plays a specified file

playFile modifier PathToFile

Here is an example of it being used:

playFile await ~/file.ezcode

In the example above, the script awaits the playing of a file by the name of 'file.ezcode.' When playing a file, any variables, objects, labels, buttons, or textboxes already created or are cretaed in the played script will be saved and can be used by the other scripts.

Here is an example of that:

FILE: root.ezcode:
  var a 10
  playFile await ~/var-modifier.ezcode
  print b
FILE: var-modifier.ezcode
  var b = 5
  b * a
OUTPUT:
  50

Here are the two modifiers that can be used:

now await

It is recommended to use the 'await' modifier instead of the 'now' modifier because of it's predictability.

Print

The 'print' function outputs a value to the console.

print hello_world

It will only output text one space infront of it.

CODE: print hello world
OUTPUT: hello

Because of these limitations, special characters were made to get around this.

New Line: \n
Space: \_
Nothing: \!

This function can also print variable:

CODE:
  var a hello\_world
  print a
OUTPUT:
  hello world

Print Raw

The 'printRaw' function outputs a value to the console.

printRaw hello world

It will any output text infront of it.

CODE: print hello world
OUTPUT: hello world

This function will display the any special characters as \n, \_, or \!. This will not print the value of a variable.

Scale

The 'scale' function is used to update the size of an object, textbox, or button.

scale controlName x y

The code above scales the specified control to the specified coordinates (x, y).

The 'x' and 'y' values represent the new scale of the control. They can be variables.

If the control is a non multi-line textbox, then the textbox will only scale on the x coordinate.

Sound

The 'sound' function is used to play, stop, or change the volume of an audio clip.

sound modifier PathToFile

The files supported are: 'WAV,' 'MP3, 'AIFF,' 'FLAC,' 'Ogg Vorbis, 'WMA,' and 'AAC'

Here are the following modifiers:

play stop volume

Play

The play modifier is used to play an audio clip.

sound play PathToFile

Stop

The stop modifier is used to stop all audio.

sound stop

Volume

The Volume modifier is used to change the audio volume. The volume value has to be withen 0 - 1.

sound volume value

Variable

The 'var' keyword is used to declare a variable.

var variableName value

The code above declares a variable named 'variableName' and assigns it the specified value.

Variables can store different types of values, such as numbers or text.

NUMBER: var a 10.1
TEXT: var a text

'var' can also be assigned the value of a variable thats already been declared

The value of a var can be changed.

var val_1 13
val_1 + 5
var val_2 val_1
val_2 / 3
print val_2
OUTPUT: 6

If the variable is text, it can also be changed.

var val_1 hello
val_1 + \_world
print val_1
OUTPUT: hello world

Text can be added to by more text, or subtracted by a number which takes away from the length of the text. Text can't be multiplied or divided

var val_1 sometext
val_1 - 4
OUTPUT: some

A variable can also be given other values. In all of the following modifiers, any value can be exchanged for a variable.

Console Input

var a ConsoleInput

This will wait for the user to input text into the consol.e

Random

var a Random min max

This will generate a random number between the minumum and maximum values.

KeyInput

var a KeyInput

This will set the value of the variable to the current key that is being pressed. If 'A' is being pressed, the variable will be set to 'A'

Sticky Key Input

var a StickyKeyInput

This will set the value of the variable to the last key that was pressed. If 'A' is being pressed, or was the last key pressed, the variable will be set to 'A'

Await Key Input

var a AwaitKeyInput

This will wait for a key to be pressed and set it to it. If 'A' is being pressed, the variable will be set to 'A'

Read File

var a ReadFile PathToFile

This will set the variable to the contents of the specified file. If this '~/' is being used, it gets the file from the directory of the current script.

From List

var a FromList listName index

This will get the value from an index of a list.

Intersects With

var a IntersectsWith object_1 object_2

This will give the variable the value of 0 or 1 if the objects are intersecting or not. 1 being intersecting and 0 being not.

From Textbox

var a FromTextbox textboxName

This will give the variable the current value of a textbox.

Raw Variable

The 'varRaw' keyword is used to define a variable but can not be assigned a value of another variable. It also cant't use any special characters (\_, \n, or \!).

varRaw name value

It can also be assigned too different modifiers. See Variable for specifics.

Write

The 'write' function writes text to a label, textbox, or button.

write controlName text

It will only write the text one space infront of it.

CODE: write controlName hello world
CONTROL'S TEXT: hello

Because of this, special characters can be used.

New Line: \n Space: \_ Nothing: \!

This function can also print variable:

CODE:
  label label_1
  var a hello\_world
  write label_1 a
LABEL'S TEXT:
  hello world

Write Raw

The 'writeRaw' function writes text to a label, textbox, or button.

writeRaw controlName text

This will write everything after the control's name. It will not use the values of variables if the text includes any names.

CODE: write controlName hello world
CONTROL'S TEXT: hello world

Write To File

The 'writeToFile' function writes a specified value to a file.

writeToFile value PathToFile

Here is an example of it being used with variables:

var a ~/file.txt
var b 63
writeToFile b a

In the example above, the variable b is written to a file named 'file.txt.' If this '~/' is used, it gets the file from the same directory as the script currently playing.

If the file doesn't exist, it will create a new file by that name.
